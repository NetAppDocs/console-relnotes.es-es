=== Recuperación ante desastres



==== La compatibilidad con Google Cloud VMware Engine (GCVE) ya está disponible de forma general (GA)

NetApp Disaster Recovery para GCVE ya está disponible de forma general (GA). Con la disponibilidad GA, Disaster Recovery para GCVE ahora es compatible con:

* *Instantáneas gestionadas por la plataforma:* Administra copias de seguridad y políticas de retención sin problemas a través de ONTAP o Google Cloud Platform (GCP).
* *Varios almacenes de datos por plan de replicación:* Simplifica tus planes de Disaster Recovery consolidando varios almacenes de datos en una única estrategia de replicación.
* *Forzar la conmutación por error para sitios de origen no disponibles:* Cambia a un sitio de recuperación de desastres si el origen no está disponible.
* *Aplazar la resincronización de SnapMirror y restaurar la replicación bajo demanda:* Aplaza las resincronizaciones de SnapMirror para evitar un impacto en las cargas de trabajo de producción durante los periodos de máxima actividad usando la opción "skip protect" y luego restaura la replicación bajo demanda con la opción re-protect


Para más información sobre el soporte para GCVE, conoce link:https://docs.netapp.com/us-en/data-services-disaster-recovery/release-notes/dr-limitations.html["configuraciones compatibles para Disaster Recovery"^].



=== Resiliencia frente al ransomware



==== Restauración limpia

Ransomware Resilience ahora es compatible con las restauraciones limpias, una nueva opción de recuperación que te ayuda a recuperarte rápida y fácilmente de los ataques de ransomware. Las restauraciones limpias ofrecen flujos de trabajo guiados que identifican los puntos de restauración óptimos para que tus cargas de trabajo vuelvan a funcionar mientras se minimiza el tiempo de inactividad. Los flujos de trabajo de la restauración limpia también proporcionan información a nivel de archivos sobre tus datos, asegurando que puedas restaurar cargas de trabajo libres de malware que evitan la reinfección y minimizan la pérdida de datos.

Para obtener información sobre las configuraciones compatibles y cómo configurar tu sistema para una restauración limpia, consulta link:https://docs.netapp.com/us-en/data-services-ransomware-resilience/task-clean-restore.html["Crear un entorno de recuperación aislado"^]. Para aprender cómo recuperar con una restauración limpia, consulta link:https://docs.netapp.com/us-en/data-services-ransomware-resilience/task-clean-restore-recover.html["Recupera cargas de trabajo con una restauración limpia"^].



==== Soporte SIEM para Splunk Enterprise

Ransomware Resilience ahora proporciona soporte de gestión de eventos de información de seguridad (SIEM) para Splunk Enterprise, lo que te permite centralizar los datos de registros y eventos y agilizar el análisis y la detección de amenazas.

Para obtener más información, consulta link:https://docs.netapp.com/us-en/data-services-ransomware-resilience/task-siem.html["Conecta NetApp Ransomware Resilience a un SIEM"^].



==== Mejora de la actividad del usuario

La información sobre las violaciones de datos se conserva ahora durante 13 meses después de un incidente, lo que permite un registro más prolongado para tus necesidades de información.

Para obtener más información, consulta link:https://docs.netapp.com/us-en/data-services-ransomware-resilience/concept-user-behavior.html["Conoce la detección de la actividad de los usuarios"^].



=== Cloud Volumes ONTAP



==== Mejora de la compatibilidad de Cloud Volumes ONTAP con las máquinas virtuales de Google Cloud C3

Cloud Volumes ONTAP ofrece ahora compatibilidad mejorada con las máquinas virtuales de la serie C3 de Google Cloud, lo que permite estas funciones adicionales:

* En tus implementaciones de Cloud Volumes ONTAP, puedes cambiar los tipos de instancia entre diferentes variantes dentro de la familia C3.
* Al cambiar el tipo de instancia, si seleccionas c3-standard-44 o c3-standard-88 y sus variantes LSSD como máquina virtual de destino, el nivel de red de salida del servicio de red de Google Tier-1 se aplica automáticamente. Si cambias a cualquier otra variante de C3, el nivel de red de salida se establece en Standard.
+

NOTE: Cambiar el nivel de la red de salida a Tier-1 conlleva costes adicionales. Planifica el despliegue y la actualización en consecuencia.

* Si seleccionas las variantes de SSD local (LSSD) de C3, Flash Cache se activa de forma predeterminada. No puedes cambiar entre las variantes de LSSD de C3 en tu implementación de Cloud Volumes ONTAP, incluso si la variante de destino admite Flash Cache.
* En tus máquinas virtuales C3, ahora puedes añadir volúmenes a agregados nuevos y existentes.
* Puedes cambiar los ajustes de IOPS y rendimiento al añadir volúmenes a través de la NetApp Console o utilizando la  `PUT /gcp/vsa/aggregates/{workingEnvironmentId}/{aggregateName}` API.

=== Cloud Volumes ONTAP



==== Posibilidad de gestionar la configuración del proxy del agente de NetApp Console para Cloud Volumes ONTAP

Ahora puedes gestionar la configuración del proxy en el agente de la NetApp Console para Cloud Volumes ONTAP, incluso si pierdes la conexión o el proxy está configurado incorrectamente. Antes, si el agente de la NetApp Console no podía conectarse a Cloud Volumes ONTAP durante 20 minutos, sobrescribía tu configuración manual del proxy con la configuración predeterminada. Esto causaba fallos de comunicación, incluyendo problemas con los mensajes de AutoSupport. Para conservar tu configuración del proxy para los sistemas existentes, ejecuta la siguiente llamada a la API:

`PUT /occm/config`

Incluye el siguiente parámetro en el cuerpo de la solicitud:

[source, json]
----
{
  "proxyMode": "No_Overwrites"
}
----
El modo predeterminado es estándar, lo que significa que el agente de la NetApp Console sobrescribe tu configuración de proxy con los valores predeterminados si no puede conectarse a Cloud Volumes ONTAP durante 20 minutos.

https://docs.netapp.com/us-en/console-automation/cvo/put-occm-config.html["Configura los parámetros modificables de NetApp Console"^]

=== Clasificación de datos



==== Versión 1.52

.Validación de recursos
Para garantizar la fiabilidad, Data Classification ahora valida la memoria disponible y los recursos de disco durante el escaneo, la generación de informes y las acciones. Cuando el uso de memoria supera el 95% o el espacio en disco cae por debajo de 6 GB, Data Classification pausa automáticamente las operaciones. Las operaciones se reanudan una vez que los recursos son suficientes.

Para monitorear tus recursos del sistema, mira link:https://docs.netapp.com/us-en/data-services-data-classification/task-health-monitor.html["Supervisa el estado de salud de Data Classification"^].

.Límite aumentado para copiar y mover
Las funciones de copiar y mover ahora son compatibles con destinos CIFS. El límite de tamaño por archivo para ambas acciones ha aumentado a 1 GB desde el límite anterior de 50 MB. Los límites de operación para ambas acciones han aumentado a 1 TB o 5 millones de archivos.

Para más información, consulta link:https://docs.netapp.com/us-en/data-services-data-classification/task-copy-move.html["Copiar, mover y borrar datos"^].

.Permisos de configuración de integración de Active Directory
Los cambios en la configuración de integración de Active Directory (AD) en Data Classification ahora están restringidos a los administradores. Los viewers ya no pueden modificar la configuración de AD.

Para obtener más información sobre la función Classification viewer, consulta link:https://docs.netapp.com/us-en/console-setup-admin/reference-iam-predefined-roles.html#data-service-roles["Roles de servicios de datos"^].

.Soporte para RHEL 9.7
Data Classification ha añadido Red Hat Enterprise Linux v9.7 para la instalación manual on-premises, incluidas las implementaciones en dark site.

=== Cloud Volumes ONTAP



==== Compatibilidad de Cloud Volumes ONTAP con la próxima generación de máquinas virtuales de Google Cloud

Con 9.18.1, NetApp realiza la transición de las nuevas implementaciones de Cloud Volumes ONTAP de las máquinas virtuales N2 a la nueva generación de máquinas virtuales de la serie C3 de Google Cloud para una experiencia más rápida y escalable. Ahora puedes aprovechar las máquinas virtuales de la serie C3 al implementar Cloud Volumes ONTAP 9.18.1 y versiones posteriores en Google Cloud. Las máquinas de la serie C3 ofrecen un rendimiento mejorado y límites de capacidad superiores mediante el uso de Google Virtual NIC (gVNIC) y discos Hyperdisk Balanced que aseguran un rendimiento dinámico para cargas de trabajo intensivas.


NOTE: Por ahora, Cloud Volumes ONTAP solo es compatible con la serie C3 en despliegues de un solo nodo.

Si tu sistema Cloud Volumes ONTAP ejecuta la versión 9.18.1 o posterior, los paquetes preconfigurados que usas para implementaciones sencillas de un solo nodo utilizan automáticamente máquinas virtuales C3, y te permiten personalizar los parámetros de IOPS y rendimiento según las necesidades de tu carga de trabajo. De manera similar, al crear agregados, puedes añadir discos Hyperdisk Balanced para lograr un mejor rendimiento y escalabilidad en Google Cloud. Además, puedes seleccionar las variantes LSSD de las máquinas de la serie C3 para compatibilidad predeterminada con Flash Cache.

No puedes cambiar el tipo de disco para las VMs C3 cuando añades volúmenes a agregados porque C3 solo admite discos Hyperdisk Balanced. De manera similar, cuando replicas un sistema con tipos de VM N2 a una VM C3, el tipo de disco se establece por defecto en Hyperdisk Balanced.

https://docs.netapp.com/us-en/cloud-volumes-ontap-relnotes/reference-configs-gcp.html["Configuraciones compatibles para Cloud Volumes ONTAP en Google Cloud"^]

https://docs.cloud.google.com/compute/docs/general-purpose-machines#c3_series["Google documentos: serie de máquinas C3"^]



==== Seguridad VNet para Cloud Volumes ONTAP en Azure

Cloud Volumes ONTAP 9.18.1 y las implementaciones posteriores en zonas de disponibilidad única y múltiple de Azure admiten el cifrado de Azure Virtual Network (VNet) como parte de su estrategia de seguridad por capas para proteger los datos en tránsito. Cloud Volumes ONTAP aprovecha el protocolo nativo Datagram Transport Layer Security (DTLS) de Azure para proteger la comunicación entre los nodos ONTAP, las interfaces de gestión y otros servicios de Azure, evitando la interceptación y el acceso no autorizado. Este cifrado a nivel de red complementa las protecciones integradas de almacenamiento y datos en reposo de ONTAP para ofrecer seguridad de extremo a extremo a tus datos.

https://review.docs.netapp.com/us-en/storage-management-cloud-volumes-ontap_gcp-sn-c3-support/reference-networking-azure.html#networking-for-azure-vnet-encryption["Redes para el cifrado de Azure VNet"^]

.Activa la clasificación de datos con Amazon FSxN para ONTAP directamente desde el canvas de la NetApp Console
Ahora puedes habilitar Amazon FSxN para sistemas ONTAP directamente desde el lienzo de NetApp Console, lo que te permite lanzar más rápido la clasificación de datos para tus sistemas FSxN.

Para más información sobre el uso de Data Classification con Amazon FSxN para ONTAP, consulta link:https://docs.netapp.com/us-en/data-services-data-classification/task-scanning-fsx.html["Escanear Amazon FSxN para volúmenes ONTAP"^].

.Aumento del límite de exportación para investigar directorios
Cuando exportas un informe de investigación sobre directorios desde el panel de control de Investigación, ahora puedes incluir 10,000 filas. Este aumento respecto al límite anterior de 5,000 filas permite investigaciones a mayor escala sobre la gobernanza y el cumplimiento de datos.

Para más información, consulta link:https://docs.netapp.com/us-en/data-services-data-classification/task-investigate-data.html["Investiga los datos"^].

.Límites aumentados para copiar y mover
Al copiar o mover archivos, ahora puedes mover archivos de hasta 250 MB, un aumento respecto al límite anterior de 50 MB.

Para más información, consulta link:https://docs.netapp.com/us-en/data-services-data-classification/task-copy-move.html["Copiar, mover y borrar datos"^].

.Visualización mejorada para pantallas de baja resolución
Data Classification ha mejorado su rendimiento de visualización para pantallas de baja resolución, mejorando la experiencia del usuario.



=== Cloud Volumes ONTAP



==== Compatibilidad con Google Cloud Infrastructure Manager

Cloud Volumes ONTAP 9.16.1 y versiones posteriores ahora admiten https://docs.cloud.google.com/infrastructure-manager/docs/overview["Google Cloud Infrastructure Manager"^] (IM) en lugar de https://docs.cloud.google.com/deployment-manager/docs/deprecations["Administrador de implementación en la nube"^] (DM) para nuevas implementaciones en Google Cloud. Google va a dejar de usar Deployment Manager como servicio de infraestructura en un futuro próximo para pasarse a Infrastructure Manager, que es más avanzado.

Ahora, despliega Cloud Volumes ONTAP para usar automáticamente Infrastructure Manager o cambia tus implementaciones existentes en Deployment Manager a Infrastructure Manager ejecutando una herramienta de conversión. La conversión es un proceso único, después del cual tus sistemas empezarán a usar Infrastructure Manager. Consulta https://docs.netapp.com/us-en/storage-management-cloud-volumes-ontap/task-gcp-dm-im-conv.html["Configura las implementaciones existentes de Cloud Volumes ONTAP para Google Cloud Infrastructure Manager"^] para ver instrucciones sobre cómo ejecutar la herramienta de conversión.

Los sistemas Cloud Volumes ONTAP que usan Infrastructure Manager usan buckets de Google Cloud Storage para almacenar datos y registros en la zona del primer despliegue para guardar registros de despliegue, que se reutilizan en despliegues posteriores. Puede que tengas costes adicionales por estos buckets, pero no edites ni elimines los buckets ni su contenido:

* `gs://netapp-cvo-infrastructure-manager-<project id>`: para las versiones de ONTAP y las plantillas Terraform de SVM usadas para nuevos despliegues de Cloud Volumes ONTAP. Dentro de esto, el bucket `dm-to-im-convert` contiene los archivos Terraform de Cloud Volumes ONTAP.
* `<gcp project number>-<region>-blueprint-config`: para almacenar artefactos de Google Cloud Terraform.

== 04 de febrero de 2026



=== Asesor digital



==== Nuevo informe de seguridad para una supervisión exhaustiva de la seguridad

Digital Advisor ahora incluye un informe de seguridad que proporciona una vista unificada de los riesgos de seguridad de ONTAP en todo tu entorno de almacenamiento. Este informe permite detectar más rápido errores de configuración, brechas de cumplimiento y problemas de seguridad en clústeres, SVMs y volúmenes, ayudándote a mantener una infraestructura de almacenamiento segura.



==== Compatibilidad mejorada de Upgrade Advisor con niveles de parches mixtos

Upgrade Advisor ahora genera informes de actualización para clústeres donde los nodos ejecutan diferentes niveles de parches dentro de la misma versión principal de ONTAP. Esta mejora asegura que los clientes puedan seguir con las actualizaciones sin problemas siempre que el clúster no esté mezclado entre versiones principales, dando más flexibilidad en la planificación de las actualizaciones.



==== Soporte ampliado para versiones de ONTAP con fin de soporte limitado

Upgrade Advisor ahora permite generar planes de actualización para las versiones de ONTAP que han alcanzado el fin del soporte limitado, con un periodo de gracia de 9 meses. Esta función ayuda a los clientes a tomar medidas a tiempo y evitar interrupciones en las actualizaciones al proporcionar una orientación clara durante el periodo de transición.

=== Keystone

Nuevas mejoras en el panel de Keystone en la NetApp Console:



==== Filtrado interactivo para los datos diarios de ráfagas acumuladas

La tabla *Ráfaga acumulada por días* de la pestaña *Tendencia de consumo* ahora admite el filtrado interactivo. Selecciona cualquier barra del gráfico de facturación de la ráfaga acumulada para ver solo los días de ese periodo de facturación. Para saber más, consulta link:./integrations/consumption-tab.html#view-daily-accrued-burst-data-usage["Ver el uso diario acumulado de datos burst"].



==== Visualización mejorada del intervalo de fechas en el gráfico de ráfagas acumuladas

El gráfico de facturación de ráfagas acumuladas en la pestaña *Tendencia de consumo* ahora muestra intervalos de fechas completos para cada periodo de facturación, como del 01 de oct de 2025 al 31 de oct de 2025, en vez de solo el mes y el año.



==== Vista ampliable de los detalles de suscripción

La pestaña *Suscripciones* incluye una opción para expandir todas las suscripciones y ver la información de los niveles de servicio de rendimiento de cada suscripción simultáneamente. Para saber más, consulta link:./integrations/subscriptions-tab.html["Ver detalles sobre tus suscripciones a Keystone"].



==== Nueva columna de tipo de uso

La pestaña *Suscripciones* incluye una columna *Tipo de uso* que muestra si la suscripción se factura en función del uso aprovisionado, físico o lógico. Para saber más, consulta link:./integrations/subscriptions-tab.html["Ver detalles sobre tus suscripciones a Keystone"].

=== Copia de seguridad y recuperación



==== Mejoras en las cargas de trabajo de volumen de ONTAP

Esta versión de cargas de trabajo de volumen de ONTAP presenta la siguiente capacidad mejorada:

*Compatibilidad con múltiples buckets*: (Vista previa privada) A partir de ONTAP 9.17.1 y versiones más recientes, ahora puede proteger los volúmenes dentro de un sistema con hasta 6 buckets por sistema en diferentes proveedores de nube.

link:https://docs.netapp.com/us-en/data-services-backup-recovery/prev-ontap-backup-manage.html["Obtenga más información sobre cómo realizar copias de seguridad de los datos del volumen de ONTAP con NetApp Backup and Recovery"].



==== Mejoras en las cargas de trabajo de VMware

Esta versión de cargas de trabajo de VMware presenta las siguientes capacidades mejoradas:

* La compatibilidad con cargas de trabajo de VMware ya está disponible de forma general (GA) en NetApp Backup and Recovery.
* Ahora puede restaurar archivos y carpetas del sistema operativo invitado.


https://docs.netapp.com/us-en/data-services-backup-recovery/br-use-vmware-restore-guest-files-and-folders-overview.html["Obtenga más información sobre cómo restaurar archivos y carpetas de invitados"].



==== Mejoras en la vista previa de las cargas de trabajo de Hyper-V

Esta versión de cargas de trabajo de Hyper-V presenta las siguientes capacidades mejoradas:

* Ahora puedes restaurar las copias de seguridad y las instantáneas de Hyper-V VM en una ubicación alternativa. Usa esta función para gestionar versiones de VM en diferentes hosts Hyper-V.
* NetApp Backup and Recovery ahora es compatible con máquinas virtuales Hyper-V aprovisionadas por System Center Virtual Machine Manager (SCVMM) y alojadas en un recurso compartido CIFS.
* Ahora puedes editar grupos de protección.



NOTE: Solo en esta versión, no es posible actualizar los complementos de NetApp para Hyper-V o Windows mediante la opción *Actualizar* en el menú Acciones. En su lugar, elimine cada host de Hyper-V y agréguelo nuevamente para actualizar los complementos.

https://docs.netapp.com/us-en/data-services-backup-recovery/br-use-hyperv-restore.html["Obtenga más información sobre cómo restaurar máquinas virtuales de Hyper-V con NetApp Backup and Recovery"].



==== Mejoras en la vista previa de las cargas de trabajo de KVM

La vista previa de cargas de trabajo KVM ahora protege los hosts KVM y las máquinas virtuales que son gestionadas por Apache CloudStack.

Para obtener detalles sobre cómo proteger las cargas de trabajo de KVM, consulte https://docs.netapp.com/us-en/data-services-backup-recovery/br-use-kvm-protect-overview.html["Descripción general de las cargas de trabajo de Protect KVM"] .
