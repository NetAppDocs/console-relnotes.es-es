=== Resiliencia frente al ransomware



==== Compatibilidad con Google Cloud NetApp Volumes (vista previa)

Ransomware Resilience ahora es compatible con los sistemas de Google Cloud NetApp Volumes, lo que te permite detectar y responder eficazmente a las amenazas de ransomware en Google Cloud NetApp Volumes. Cuando realizas la detección, Ransomware Resilience muestra automáticamente las cargas de trabajo de los sistemas de Google Cloud NetApp Volumes. Cuando descubres los sistemas, aparecen en el panel de Ransomware Resilience. El soporte para Google Cloud NetApp Volumes en Ransomware Resilience está actualmente en vista previa.

Para obtener más información, consulta link:https://docs.netapp.com/us-en/data-services-ransomware-resilience/concept-ransomware-resilience.html["Conoce la resiliencia ante ransomware"^]. Para obtener información sobre el alcance de la asistencia de Google Cloud NetApp Volumes, consulta link:https://docs.netapp.com/us-en/data-services-ransomware-resilience/rp-reference-limitations.html["Limitaciones de la resistencia al ransomware"^].



==== Mejoras en la restauración limpia

La restauración limpia ahora admite informes. Con los informes, puedes revisar y evaluar las restauraciones limpias, guardando esta información sin conexión como un archivo CSV.

La restauración limpia también permite ahora detener el análisis. Si detienes el análisis, debes iniciar de nuevo la restauración limpia.

Para saber más sobre informes, consulta link:https://docs.netapp.com/us-en/data-services-ransomware-resilience/rp-use-reports.html["Descargar informes en Ransomware Resilience"^]. Para saber más sobre recuperación, consulta link:https://docs.netapp.com/us-en/data-services-ransomware-resilience/task-clean-restore-recover.html["Recupera cargas de trabajo con una restauración limpia"^].



=== Cloud Volumes ONTAP



==== Implementación simplificada de Cloud Volumes ONTAP en Azure

Ahora puedes usar Quick create para implementar Cloud Volumes ONTAP en Azure tanto para configuraciones de nodo único como de alta disponibilidad (HA). Este proceso optimizado reduce el número de pasos en comparación con el método avanzado, establece automáticamente los valores predeterminados en una sola página y minimiza la navegación, lo que hace que la implementación sea más rápida y sencilla.


NOTE: Si usas Creación rápida para desplegar un nuevo sistema Cloud Volumes ONTAP en Azure, la consola NetApp selecciona las credenciales de Azure por defecto. Puedes desplegar todos tus sistemas Cloud Volumes ONTAP usando las credenciales iniciales de Azure. Si quieres usar otra suscripción y/o credenciales, necesitas cambiar al despliegue en modo avanzado. Esta limitación se solucionará en una versión futura.

=== Cloud Volumes ONTAP



==== Facturación flexible para Cloud Volumes ONTAP y servicios de datos en AWS

NetApp ofrece opciones de facturación mejoradas para tus suscripciones a Cloud Volumes ONTAP en AWS, dándote más control sobre cómo pagas la capacidad de Cloud Volumes ONTAP y los servicios de datos de NetApp, como BlueXP backup and recovery. Antes, todo el uso de Cloud Volumes ONTAP y los servicios de datos se facturaba junto a través del listado NetApp Intelligent Services (NIS) en el AWS marketplace.

Debido a los cambios en AWS Enterprise Discount Program (EDP), NetApp ha introducido una suscripción independiente a Cloud Volumes ONTAP en el marketplace de AWS que te permite separar la facturación de la capacidad de Cloud Volumes ONTAP de los servicios de datos. Esta separación te da más flexibilidad para gestionar tus créditos y costos de EDP. NetApp recomienda usar la suscripción independiente para una mejor gestión de los costos de tu uso de Cloud Volumes ONTAP, mientras sigues usando la suscripción NIS para la facturación de los servicios de datos después del 31 de julio de 2026, cuando entren en vigor los requisitos de EDP.

La suscripción independiente a Cloud Volumes ONTAP en AWS solo admite paquetes de licencia Essentials o Freemium basados en capacidad válidos. No se admite el paquete Professional. Puedes seleccionar el método de facturación que mejor se adapte a tus necesidades:

* *Suscripción independiente a Cloud Volumes ONTAP (recomendada)*: Paga el uso de Cloud Volumes ONTAP a través de la suscripción independiente en el marketplace de AWS, mientras sigues facturando los servicios de datos a través de la suscripción NIS. Esta es la opción recomendada para la mayoría de los clientes que tienen tanto suscripciones independientes como NIS asociadas a su cuenta de AWS en la NetApp Console.
* *Suscripción NIS combinada*: Paga tanto Cloud Volumes ONTAP como los servicios de datos juntos a través del listado NIS existente. El enfoque combinado será válido para los clientes existentes que solo tengan suscripciones NIS asociadas a su cuenta de AWS en la NetApp Console. Sin embargo, NetApp recomienda que configures tu preferencia de facturación para tu uso de Cloud Volumes ONTAP y cambies a la suscripción independiente para una mejor gestión de costos.


Dependiendo de tu tipo de suscripción, obtienes estos escenarios:

.Sólo suscripción independiente (nuevos clientes)
Si solo tienes una suscripción independiente de AWS asociada a tu cuenta de AWS en la NetApp Console y no tienes suscripciones de NIS, solo puedes seleccionar el paquete de licencias Essentials durante la implementación. La opción de servicios de datos *Backup and Recovery* no está disponible para ti.

.Tanto las suscripciones independientes como las de NIS (clientes actuales)
Si tienes suscripciones NIS y autónomas asociadas a tu cuenta de AWS, y no has establecido una preferencia de facturación a través de la pestaña *Preferencias de facturación* en la NetApp Console, el uso de Cloud Volumes ONTAP sigue facturándose a través de NIS de forma predeterminada. Esta también es la opción predeterminada para cualquier instancia de Cloud Volumes ONTAP que ejecute el paquete Professional, porque el listado autónomo no admite Professional. Puedes establecer una preferencia de facturación y elegir cómo facturar cada servicio:

* Establece la suscripción independiente como la cuenta de facturación preferida para Cloud Volumes ONTAP (con el paquete Essentials). Si solo tienes el paquete Professional, puedes convertirlo en un paquete Essentials y cobrar tu uso de Cloud Volumes ONTAP configurando la preferencia de facturación. Te recomendamos este enfoque porque te ayuda a gestionar tus créditos EDP de Cloud Volumes ONTAP de manera más eficaz.
* Utiliza la suscripción NIS para facturar los servicios de datos o el uso de Cloud Volumes ONTAP a través del paquete Professional. Durante el despliegue, activa el servicio de datos *Backup and Recovery* y selecciona el paquete de licencia Professional para facturar los servicios de datos a través de la suscripción NIS.


.Sólo suscripción a NIS (clientes actuales)
Esto es aplicable si eres un cliente existente con suscripciones NIS asociadas a la cuenta de AWS. Puedes seguir facturando tanto Cloud Volumes ONTAP como los servicios de datos juntos, tal como lo haces ahora. Sin embargo, NetApp recomienda cambiar al enfoque de suscripción independiente para una mejor gestión de los costes de tu uso de Cloud Volumes ONTAP:

* Suscríbete a un nuevo listado independiente en el portal de AWS.
* Vincula la suscripción independiente a tu cuenta de AWS.
* Establece tu preferencia de facturación en la suscripción independiente para el uso de Cloud Volumes ONTAP con el paquete Essentials.
* (Si procede) Convierte tu paquete Professional en un paquete Essentials y establece la preferencia de facturación.
* Continúa facturando servicios de datos a través de tu suscripción NIS.



NOTE: No se duplica la medición cuando usas varios tipos de suscripción o migras entre listados.

=== Clasificación de datos



==== Página de configuración mejorada

Data Classification ha actualizado y mejorado la página de Configuración, permitiendo una experiencia de uso mejorada para gestionar ajustes, sistemas, habilitar el escaneo y supervisar la salud del sistema. La nueva página de Configuración proporciona la funcionalidad existente en un formato más fácil de usar, además de la capacidad de filtrar sistemas por tipo, estado de error y estado de escaneo. La nueva página también proporciona un resumen de los sistemas conectados y vistas generales de los escaneos activos y configurados.

Revisa un link:https://youtu.be/_8YJ3KyZ7nI["video tutorial"^]. Para más información, consulta link:https://docs.netapp.com/us-en/data-services-data-classification/task-scanning-overview.html["Escanea las fuentes de datos con Data Classification"^].



==== Función IAM de administrador de conformidad

Clasificación de datos ahora admite un rol de administrador de cumplimiento para gestionar los permisos en Clasificación de datos. El rol de administrador de cumplimiento no tiene acceso a recursos, incluidos los agentes de Console y los espacios de trabajo. Los usuarios con el rol de administrador de cumplimiento deben estar asociados con al menos otro rol.

Al instalar Data Classification, debes tener el rol de administrador de la plataforma de Console o el rol de administrador de Compliance.

Si eres un usuario existente, el acceso debería continuar normalmente. Por ejemplo, un rol de plataforma que tiene un rol de administrador de carpeta o proyecto tiene acceso existente.

Para más información, consulta link:https://docs.netapp.com/us-en/console-setup-admin_classification-role/reference-iam-data-classification.html["Funciones de clasificación de datos"^].



==== Implementa Data Classification con OVA

NetApp Data Classification ahora admite la implementación con VMware Open Virtual Appliance (OVA), lo que ofrece una estrategia de implementación simplificada para entornos VMware vSphere locales.

Para más información, consulta link:https://docs.netapp.com/us-en/data-services-data-classification/task-deploy-ova.html["Implementa Data Classification con OVA"^].



=== Recuperación ante desastres



==== Compatibilidad con cargas de trabajo basadas en Kubernetes (vista previa)

NetApp Disaster Recovery ahora incluye compatibilidad con cargas de trabajo basadas en Kubernetes. Con la compatibilidad con Kubernetes, puedes proteger aplicaciones que se ejecutan en clústeres de Kubernetes, incluidos los entornos de virtualización OpenShift. Disaster Recovery es compatible con link:https://docs.netapp.com/us-en/trident/trident-protect/trident-protect-requirements.html["cualquier versión de Kubernetes actualmente soportada por Trident Protect"^], lo que garantiza una integración fluida y sólidas capacidades de recuperación ante desastres para aplicaciones en contenedores.

Funciones clave del soporte de cargas de trabajo basado en Kubernetes:

* **Amplia compatibilidad**: Disaster Recovery es compatible con todas las versiones de Kubernetes compatibles con Trident Protect, incluida Red Hat OpenShift Virtualization.
* **Protección centrada en las aplicaciones**: Ahora puedes proteger las cargas de trabajo de Kubernetes a nivel de aplicación, asegurando la recuperación de los servicios críticos durante eventos de conmutación por error.
* **Orquestación simplificada**: Disaster Recovery ofrece flujos de trabajo intuitivos para automatizar la copia de seguridad y la replicación de las cargas de trabajo de Kubernetes, aprovechando la misma tecnología eficiente SnapMirror utilizada para entornos VMware.


La compatibilidad con cargas de trabajo basadas en Kubernetes se encuentra actualmente en fase de vista previa. Para empezar a proteger las cargas de trabajo basadas en Kubernetes, consulta link:https://docs.netapp.com/us-en/data-services-disaster-recovery/get-started/task-kubernetes-requirements.html["Requisitos de Kubernetes para NetApp Disaster Recovery"^].



=== Resiliencia frente al ransomware



==== Mejoras en la restauración limpia

La restauración limpia ahora admite la recuperación en una ubicación alternativa, lo que te permite restaurar volúmenes en diferentes sistemas, máquinas virtuales de almacenamiento y agregados gestionados por el mismo agente de Console. Además, la restauración limpia ha eliminado la limitación de siete días en la búsqueda de instantáneas viables para la recuperación. La restauración limpia ahora continúa buscando en las instantáneas disponibles un punto de recuperación viable independientemente de la distancia desde el evento de ransomware.

link:https://docs.netapp.com/us-en/data-services-ransomware-resilience/task-clean-restore-recover.html["Recupera cargas de trabajo con una restauración limpia"^].



==== Google SecOps soporte SIEM

Ransomware Resilience ahora es compatible con la gestión de información y eventos de seguridad (SIEM) con Google SecOps, lo que te permite enviar datos automáticamente a Google SecOps para agilizar el análisis y la detección de amenazas.

Para obtener información sobre la configuración, consulta link:https://docs.netapp.com/us-en/data-services-ransomware-resilience/task-siem.html["Conecta NetApp Ransomware Resilience a un SIEM"^].



==== El soporte de Azure NetApp Files ya está disponible de forma general

Ransomware Resilience ahora es compatible con los sistemas Azure NetApp Files, lo que te permite detectar y responder eficazmente a las amenazas de ransomware en Azure NetApp Files. Cuando descubres cargas de trabajo, Ransomware Resilience ahora presenta Azure NetApp Files y las muestra en el panel de protección. El soporte de Ransomware Resilience para Azure NetApp Files incluye estrategias de detección y protección solo con instantáneas. El soporte para Azure NetApp Files ya está disponible de forma general (GA).

Para más información, consulta link:https://docs.netapp.com/us-en/data-services-ransomware-resilience/concept-ransomware-resilience.html["Conoce la resiliencia ante ransomware"^].



==== Soporte SOAR para bloquear usuarios

Ransomware Resilience ahora soporta el bloqueo de usuarios individuales con Google SecOps y Splunk SOAR playbooks. Con esta adición, puedes automatizar el bloqueo de usuarios cuando Ransomware Resilience detecta actividad sospechosa de un usuario, como una violación de datos o la eliminación masiva de datos.

link:https://docs.netapp.com/us-en/data-services-ransomware-resilience/reference-soar.html["Integra un playbook SOAR"^].

=== Cloud Volumes ONTAP



==== Posibilidad de gestionar la configuración del proxy del agente de NetApp Console para Cloud Volumes ONTAP

Ahora puedes gestionar la configuración del proxy en el agente de la NetApp Console para Cloud Volumes ONTAP, incluso si pierdes la conexión o el proxy está configurado incorrectamente. Antes, si el agente de la NetApp Console no podía conectarse a Cloud Volumes ONTAP durante 20 minutos, sobrescribía tu configuración manual del proxy con la configuración predeterminada. Esto causaba fallos de comunicación, incluyendo problemas con los mensajes de AutoSupport. Para conservar tu configuración del proxy para los sistemas existentes, ejecuta la siguiente llamada a la API:

`PUT /occm/config`

Incluye el siguiente parámetro en el cuerpo de la solicitud:

[source, json]
----
{
  "proxyMode": "No_Overwrites"
}
----
El modo predeterminado es estándar, lo que significa que el agente de la NetApp Console sobrescribe tu configuración de proxy con los valores predeterminados si no puede conectarse a Cloud Volumes ONTAP durante 20 minutos.
